# AI 热点短视频文案流水线

## 执行速查

1. **热点主线**：默认调用同仓库安装的 `aihot` 获取 AI HOT 热点，不凭记忆写近期热点。
2. **普通请求**：用户只要 AI 热点文案时，直接走默认主线，不追问账号方向。
3. **定制请求**：用户明确要“按账号方向/定位/赛道”来做时，先生成本次账号方向配置，再回到主线。
4. **所有输出**：每条都必须有 `口播稿` 和 `字幕版`。字幕版一句一行，方便复制到剪辑软件。
5. **可看性优先**：热点不是选题。先判断它有没有停留理由、可看点、用户关联和信息回报，再决定是否生成。
6. **硬边界**：不编造事实；不做真人出镜；不做录屏实操；不暴露底层内容方法工具名。

## 默认参数

| 项目 | 默认值 |
|---|---|
| 热点来源 | AI HOT 精选条目，由同仓库安装的 `aihot` 获取 |
| 时间窗口 | 最近 24-72 小时，按热点密度调整 |
| 批量数量 | 5 条可用选题 |
| 发布平台 | 抖音、视频号 |
| 视频形态 | 数字人口播 + AI 画面 |
| 视频时长 | 70-85 秒 |
| 默认观众 | 对 AI 有兴趣，但没有耐心看行业简报的短视频用户 |
| 默认方向 | 有停留理由的观点、趋势判断、反常识、争议讨论、故事化解读 |
| 禁止方向 | 真人实拍、录屏实操、逐步点击教程、纯新闻复述 |

## 能力组成

### 必装热点能力

本仓库发布时应同时安装两个 Skill：

- `aihot`：负责从 AI HOT 获取近期 AI 热点、精选条目、分类、关键词结果或日报。
- `aihot-video-copy-skill`：负责把热点筛选、改写和编排成短视频文案。

执行时默认先调用 `aihot`。如果 `aihot` 缺失，不要跳过热点源直接写文案，要提示用户重新运行组合安装命令，或让用户提供热点链接、标题、文章素材。

如果 `aihot` 暂时不可用或返回内容不足，只能在用户确认后再联网搜索。确认前不要主动搜索，不要凭记忆编造近期热点。

### 内置内容方法

本 Skill 内置完成以下内容方法，不要求用户额外安装其它内容类 Skill：

- 选题诊断：判断热点是否有短视频价值、观众价值、事实来源和 AI 画面表达空间。
- 可看性诊断：判断热点是否有停留理由、可看点、用户关联和信息回报；内部评分不强制输出。
- 表达诊断：检查文案是否像数字人口播，而不是文章、新闻稿或工具说明书。
- Hook 优化：检查前 5 秒是否能独立吸引人，不依赖标题或封面。
- 标题生成：生成适合抖音/视频号的标题候选，保留悬念，不夸大事实。
- AI 写作特征检查：定位模板感、书面腔、过度总结、概念堆叠和虚假顺滑。
- 字幕切分：把口播稿切成一句一行，方便复制到剪辑软件。

内置方法的原则：先把事情讲清楚，再追求表达漂亮；只改表达，不改事实；宁可跳过信息不足的热点，也不编造细节。

### 外部能力可选增强

如果当前环境里存在更完整的同类内容方法能力，可以借用来增强诊断、Hook、标题或 AI 写作特征检查，但最终输出必须服从本 Skill 的格式和质量标准。

用户可见输出里只写“内容诊断”“开头优化”“标题优化”“AI 写作特征检查”等功能名，不暴露底层工具名、内部前缀或实现来源。

## 流程

### 0. 判断是否进入账号定制

默认不进入。只有用户明确表达以下意图时才进入：

- 我想要定制化匹配我的账号内容方向
- 按我的账号定位来生成
- 我的账号是面向普通人的 AI 科普号
- 我主要做 AI 热点资讯
- 我的账号做 AI 工具轻教程，不做录屏
- 这批文案要贴合我的账号赛道

如果用户想定制但没说账号方向，只追问一次：

```text
为了让这批文案贴合你的账号，我需要先知道你的账号主要讲什么方向。

你用一句话说就行，比如：面向普通人的 AI 科普、AI 热点资讯、AI 工具轻教程、商业机会解读、工具测评。

这个方向会影响我选哪些热点、用什么角度写，以及字幕节奏怎么切。
```

如果意图有点模糊，用普通中文做短确认，不写英文术语或内部概念：

```text
我理解你是想让这批文案更贴合“AI 工具教程号”的方向。

这会影响我选哪些热点、用什么角度写，以及字幕怎么切。

如果按账号方向做，我会优先选能讲方法和使用场景的热点。
如果不需要定制，我就按默认热点文案流程走。
```

定制只影响本次生产，不保存账号配置，不建立账号库，不让用户填长表。

账号方向配置包含：

- 账号方向
- 目标观众
- 内容风格
- 禁止方向
- 选题筛选逻辑

### 1. 获取热点

使用同仓库安装的 `aihot` 获取实时资料，不凭记忆写热点。默认用精选条目；只有用户明确要“全部”“日报”“某分类”“某关键词”时才切换。

如果 `aihot` 缺失，停止生成并输出：

```text
当前缺少 AI Hot 热点获取能力，不能直接生成近期热点文案。

请重新运行组合安装命令，或直接发给我一个 AI 热点链接、标题、文章素材，我可以基于你提供的素材继续生成文案。
```

如果 `aihot` 可用但结果不足，先调整时间窗口、分类或关键词一次。仍不足时，询问用户：

```text
当前 AI Hot 获取到的信息不足。

我可以继续联网搜索近期 AI 热点，再帮你筛选适合短视频的选题。

你确认后我再开始搜索。
```

用户确认前，不主动联网搜索。

每条热点保留：

- 原始标题
- 来源名称
- 原文链接
- 发布时间
- 分类
- 摘要或关键事实

用户可见结果不要暴露 API 参数、游标、限流或调用细节。

### 2. 筛选选题

先做可看性四问：

| 问题 | 作用 |
|---|---|
| 这件事有什么变化？ | 避免只复述新闻 |
| 这件事有什么可看点？ | 找到停留理由 |
| 这件事和用户有什么关系？ | 建立观看动机 |
| 看完能得到什么判断？ | 提供信息回报 |

如果四个问题里只能回答一个，默认跳过，不强行生成。

再做内部可看性评分，评分不强制输出给用户：

| 维度 | 分值 | 判断 |
|---|---:|---|
| 停留理由 | 0-2 | 前 5 秒有没有继续看的理由 |
| 新鲜变化 | 0-2 | 有没有明确变化、反差或趋势 |
| 用户关联 | 0-2 | 能不能落到用户关心的场景 |
| 信息回报 | 0-2 | 看完能不能得到一个判断 |
| 表达空间 | 0-2 | 能不能用口播和 AI 画面讲清楚 |

8-10 分优先生成；6-7 分可以生成但必须强化 Hook 和角度；5 分及以下默认跳过或换选题。

保留至少满足 3 条的热点：

- 有变化、冲突、趋势或反常识点
- 有停留理由、利益、风险、场景代入或明确判断
- 非专业观众能理解影响
- 有来源和事实，能避免编造
- 能用 AI 画面表达
- 不依赖真人、录屏或软件逐步操作
- 能形成观点，而不只是新闻复述

剔除或降级：

- 没有可追溯来源
- 只是技术更新日志
- 必须真人演示或录屏操作
- 需要编造数据、日期、公司动作或产品能力

账号定制时，再按账号方向筛选：

| 账号方向 | 优先保留 |
|---|---|
| 普通人科普 | 能翻译成普通人影响、低术语的话题 |
| 热点资讯 | 新鲜、重要、有来源、适合当下解释的话题 |
| 轻教程方法 | 能变成使用场景、方法框架或 3-5 个轻步骤的话题 |
| 商业机会 | 改变成本、流程、机会或风险的话题 |
| 工具测评 | 有明确功能变化，能判断适合谁的话题 |

### 3. 生成切入角度

每条入选热点都输出：

- 表面事实：发生了什么
- 可看点：为什么这条热点值得做成短视频
- 核心角度：为什么值得讲，不只是复述新闻
- 目标观众：谁该看
- 用户关联：和目标用户有什么关系
- 观看悬念：为什么要继续看
- 内容承诺：看完能明白什么
- 核心判断：看完这条视频应该得到什么判断

冲突和反差是优先抓手，但不是唯一抓手。也可以用利益、风险、场景代入、新鲜变化、明确判断制造可看性；不要为了反差制造假冲突。

账号定制时按方向改角度：

- 普通人科普：这和普通人有什么关系
- 热点资讯：发生了什么，为什么现在重要
- 轻教程方法：可以怎么用，但不做录屏步骤
- 商业机会：成本、流程、机会或风险变了什么
- 工具测评：适合谁，不适合谁，值不值得试

### 4. 生成口播稿

每条生成 70-85 秒中文口播稿，约 360-520 个中文字符，以真实口播节奏为准。

默认结构：

1. `0-5 秒`：先给停留理由，独立成立，不依赖标题或封面。
2. `5-15 秒`：只交代必要的可追溯事实。
3. `15-30 秒`：讲清表面看点和真正看点。
4. `30-55 秒`：落到用户场景，说明和目标观众的关系。
5. `55-75 秒`：给出判断，讲清现在该怎么看。
6. `75-85 秒`：用一句有力的话收束，不用空泛鸡汤。

账号定制时使用对应骨架：

| 账号方向 | 文案骨架 |
|---|---|
| 普通人科普 | Hook → 事实 → 白话解释 → 普通人影响 → 判断 |
| 热点资讯 | Hook → 谁做了什么 → 关键变化 → 短期影响 → 为什么值得关注 |
| 轻教程方法 | Hook → 使用场景 → 3 个轻步骤/思路 → 适合谁/不适合谁 → 下一步 |
| 商业机会 | Hook → 事实 → 成本/流程变化 → 谁能借势 → 不要误判什么 |
| 工具测评 | Hook → 变了什么 → 谁该关注 → 谁不用急 → 值不值得试 |

写作要求：

- 像数字人能直接念出来的话。
- 句子短，少书面腔。
- 一个句子最多放一个密集术语。
- 前 5 秒必须包含关系、利益、风险、新鲜变化、悬念或判断中的至少两项。
- 不使用“今天我们来聊”“最近 AI 圈发生大事”“AI 行业迎来重大变化”这类无停留理由的开头。
- 不编造事实、数据、引语或产品能力。
- 教程只做轻方法，不写点击路径、界面操作或录屏脚本。

### 5. 诊断和优化

先用本 Skill 内置内容方法完成诊断和优化；如果环境里有同类增强能力，可以借用，但不要在最终输出里暴露工具名。

按顺序检查：

1. 选题诊断：这个热点是否值得讲，是否有变化、冲突、趋势或反常识点。
2. 可看性诊断：前 5 秒是否有停留理由，是否有可看点、用户关联和信息回报。
3. 结构诊断：口播是否按 Hook、事实、解释、影响、判断推进。
4. 开头优化：前 5 秒是否独立成立，是否有话题、抓手和可信度。
5. 标题优化：至少生成 3 个标题，覆盖认知冲突、好奇缺口、身份代入、数字锚定等不同角度。
6. AI 写作特征检查：定位模板感、书面腔、概念堆叠、连接词过多、结尾鸡汤、句子过于均匀等问题。

枯燥文案按以下规则反向改写：

| 枯燥类型 | 改写方向 |
|---|---|
| 新闻复述 | 改成“这件事真正影响谁” |
| 技术介绍 | 改成“用户能少做什么、多得到什么” |
| 行业简报 | 改成“这件事暴露了什么变化” |
| 平铺直叙 | 改成“先给判断，再讲事实” |
| 术语堆叠 | 改成“一句术语 + 一句白话解释” |
| 空泛总结 | 改成“一个明确判断或提醒” |
| 工具说明书 | 改成“使用场景 + 适合谁/不适合谁” |

处理规则：

- 只改具体问题句，保留事实和原意。
- 不为了“去 AI 味”而降低信息密度。
- 不把文案改成段落文章，要保留口播停顿。
- 不编造数据、来源、时间、引语或产品能力。
- 如果选题本身太薄或改完仍然枯燥，优先换选题，不强行优化。

### 6. 生成 AI 画面方向

每条输出 5-8 个可用于 AI 视频或图片生成的画面节拍。

要求：

- 跟口播时间线一致
- 不依赖真实产品界面，除非来源里有
- 优先用概念画面、信息图、字幕卡、场景模拟
- 适合批量生成

账号定制时画面方向：

- 普通人科普：类比画面、概念可视化
- 热点资讯：新闻感、时间线、关键词字幕
- 轻教程方法：步骤卡片、使用场景、简单流程图
- 商业机会：流程图、成本变化、机会地图
- 工具测评：对比卡片、适用/不适用人群

### 7. 生成字幕版

每条口播稿后必须输出字幕版。字幕版用于复制到剪辑软件，不能是整段。

规则：

- 一行一句，或一行一个自然停顿
- 每行通常 8-18 个中文字
- 单行不超过 24 个中文字，专有名词除外
- 按口播节奏切，不按书面段落切
- 一行只表达一个清楚意思
- 不用项目符号
- 和口播稿事实一致

示例：

```text
这次 AI 更新
真正重要的不是功能变多了
而是它开始接近真实工作流

以前你要自己找资料
自己整理表格
自己复制到不同软件里
```

### 8. 质量检查

每条输出前检查：

- 有来源链接
- 事实能追溯到热点资料
- 口播稿适合 70-85 秒
- 有明确可看点，不只是新闻复述
- 前 5 秒有停留理由
- 有新鲜变化、利益、风险、场景代入或明确判断
- 看完能得到一个判断
- Hook 不依赖标题或封面
- 不需要真人、实拍、录屏或软件操作画面
- 表达不像行业简报、新闻稿、报告或 AI 总结
- 至少 3 个标题候选
- 已检查并处理 AI 写作特征
- 有一句一行字幕版，且和口播稿一致

账号定制时追加检查：

- 说明为什么适合当前账号方向
- 文案骨架符合账号方向
- Hook 和标题符合目标观众理解门槛
- 画面方向符合账号内容形态
- 教程类不依赖录屏
- 商业类不给无依据赚钱承诺
- 科普类不堆术语
- 资讯类有来源和判断
- 测评类不假装实测

不达标则改一次；仍不达标就跳过并说明原因。

## 异常处理

- 缺少 `aihot`：停止生成，提示重新运行组合安装命令，或让用户提供热点素材。
- 热点太少：调整时间窗口、分类或关键词一次；仍不足就说明不足。
- 需要 AI Hot 之外的信息：先问用户是否允许联网搜索；用户确认后才执行。
- 事实太薄：不猜，不扩写，标为“信息不足”。
- 选题太技术化：尝试转成观众影响；不成立就跳过。
- 用户要求真人出镜或录屏：说明本流程只做数字人口播 + AI 画面，并给兼容写法。
- 批量超过 10 条：建议分批，避免质量下降。

## 完成标准

- 已通过 AI Hot 获取热点；如果缺少热点能力，已说明原因并给出组合安装或用户供稿路径。
- 每条入选内容都有来源、角度、目标观众、Hook、口播稿、字幕版、画面方向、标题候选、AI 写作特征检查。
- 字幕版一句一行，能直接复制到剪辑软件。
- 文案适合数字人口播 + AI 画面。
- 没有编造或扭曲事实。
- 跳过的选题有原因。
- 用户可见文本不暴露底层内容方法工具名或内部前缀。
