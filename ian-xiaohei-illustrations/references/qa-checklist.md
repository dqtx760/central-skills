# QA Checklist

## 必过项

- 是 16:9 横版。
- 背景是干净白底。
- 有 KATONG。
- KATONG 角色特征完整：短黑寸头、浓眉细眼、黑hoodie、皇家蓝裤子 #4A78F0、胸前白色 34 号徽章。
- KATONG 承担核心动作，不只是装饰。
- 画风是极简扁平矢量、干净轮廓线，不是手绘 / 动漫 / 3D / 写实。
- 没有复刻旧案例构图，而是为当前文章生成了新隐喻。
- 画面干净、专业、现代。
- 主体不超过画面约 60%。
- 一张图只讲一个核心结构。
- 中文标注少、短、能读，用无衬线字体。
- 皇家蓝 #4A78F0 用于角色裤子和主路径 / 箭头。
- 红色只用于重点、问题、提醒或结果。
- 蓝色只用于补充说明、反馈或系统状态。

## 失败信号

出现以下情况，重生成或局部编辑：

- 左上角有“常见坑 / Workflow / 系统架构图 / 路线图”等标题。
- KATONG 角色特征漂移：丢了徽章 34、换了发色 / 发型、穿了别的衣服、画风变 anime / 3D / 写实。
- KATONG 像吉祥物、表情包或可爱卡通。
- 画面像 PPT、课程课件、正式流程图。
- 元素太多、箭头太多、节点太多。
- 文字变成大段解释。
- 背景有纸纹、阴影、渐变、米色、噪点。
- 真实 UI 截图或科技感界面。
- 中文错字严重或标注不可读。
- 画面出现手绘抖动线条、纸张感（本 skill 已切换为矢量风）。
- 和 `assets/examples/` 里的旧案例构图过于相似。

## 迭代方法

- 太普通：让 KATONG 成为动作主体，加入一个成立的隐喻。
- 角色漂移：用 prompt-template.md 里的「修正角色一致性」编辑提示，做局部 edit。
- 太复杂：删节点，只保留一个动作和 3-5 个短标注。
- 太可爱：强调 calm confident expression、not cute、not mascot、minimalist flat vector。
- 太 PPT：去掉标题、边框、整齐网格和过多箭头，改成有行动力的场景。
- 太像旧案例：保留核心意思，换掉主物件和 KATONG 的动作。
- 文字错：优先局部编辑；错得多就重生成并减少标注数量。

## 交付判断

高质量图应该让读者第一眼觉得干净专业，然后 1 秒内看懂结构。

如果第一眼像教程页或企业 PPT 老式信息图，而不是一张现代干净的品牌矢量插画，就不合格。
