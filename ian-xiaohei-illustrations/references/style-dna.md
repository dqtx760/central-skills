# 风格 DNA

## 一句话

极简扁平矢量、干净轮廓、企业插画质感、克制配色、留白、结构清楚但不说明书。

像一个现代创业公司用作品牌配图的、精致但不油腻的几何矢量插画。

## 必须

- 16:9 横版正文配图。
- 纯白或极浅背景：不要米色、纸张纹理、渐变、阴影、噪点、复古纸感。
- 极简扁平矢量风格（minimalist flat vector）：几何形体、干净轮廓线（clean outlines）、色块分明。
- 大量留白：主体占画面约 40%-60%，至少 35% 空白，最好有一整块安静区域。
- 少量中文标注：最多 5-8 处，每处尽量 2-8 个字，用清晰的无衬线字体，不要手写体。
- 一张图只讲一个核心动作、结构、状态或隐喻。
- 结构要自然表达，不要在图上写结构类型名称。

## 颜色

- 主色 黑：角色 hoodie、主体物件、轮廓线、主要文字。
- 品牌蓝 皇家蓝 #4A78F0：角色裤子、主流程、路径、箭头、自动化流向、从 A 到 B 的移动关系。
- 强调红：只用于重点批注、问题、情绪点、关键提醒、结果。
- 辅助蓝（浅蓝）：补充说明、脑内状态、系统状态、第二层解释、AI/assistant/自动化提示。
- 中性灰：次要结构、边框、辅助线。

蓝色不每张都必须全用。颜色要克制，宁可少不要多。整体配色保持「企业品牌插画」的干净质感。

## 绝对不要

- 不要 anime / manga / 日漫风。
- 不要 Pixar / 3D render / 三维渲染。
- 不要 photorealistic / 写实照片风。
- 不要可爱卡通、Q 版、儿童插画。
- 不要复杂架构图、密集节点图。
- 不要 PPT 信息图那种死板的方框流程。
- 不要课程课件风。
- 不要科技感 UI、真实 App 截图。
- 不要复杂背景、渐变、阴影、纹理、噪点。
- 不要手绘线稿、抖动笔触、纸张感（本 skill 已从手绘风切换到矢量风）。
- 不要把每个节点都解释清楚。
- 不要左上角写「Workflow 流程图 / 系统架构图 / 常见坑 / 路线图」等类型标题。

## 审美方向

要精致、干净、专业、有行动力、现代创业气质。

不要怪诞、荒诞、冷幽默（这些是旧版小黑手绘风的特征，现已废弃）。也不要过度严肃、死板、企业 PPT 感。
