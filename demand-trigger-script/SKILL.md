# 需求引爆点 · 高转化短视频脚本技能

基于"需求引爆点"逻辑撰写短视频文案。核心原则：在对的人处于"我必须现在解决"的引爆点那一刻，出现在他面前。

## 核心逻辑（来源原文）

需求引爆点 = 用户从"我有这个需求"到"我必须现在解决"的那一瞬间

在此之前刷到你 -> 收藏
在此之后刷到你 -> 直接付费

### 精准客户画像

精准客户有一个共同特征：**已经为这件事押进去了筹码**（花了钱、耗了时间、错过了机会），但结果还没出来。

这种人有一个强烈的内驱力：**不想让之前的投入全打了水漂**。

他们找的不是方案，是**止损的出口**。

### 内容触发开关

| 开关 | 人群 | 决策路径 | 内容策略 |
|------|------|----------|----------|
| **欲望** | 被画面击中的人 | 画面 -> 自己找理由 -> 下单 | 给出极度具体的"拥有之后"的状态，越清晰越有力 |
| **不安** | 需要确定感的人 | 买的是确定感，不是产品 | 传递信号：跟着我这件事稳 |

> 光靠逻辑说服不了高付费，光靠情绪撑不起复购。**两个开关组合用**，才能让用户在下单那一刻不打折扣。

### 核心误区

```
[X] 追播放量 -> 播放量是平台的情绪奖励，跟签单无关
[V] 有效获客 -> 每发一条都在命中一批准备好了的买单者

账面比数据重要，先打通一单再探放量。
```

## 执行流程

### Step 1：分析选题，定位精准客户

拿到选题后，先回答以下问题：

| 问题 | 作用 |
|------|------|
| 这个选题对应谁的需求？ | 明确目标人群 |
| 他们为这件事已经押进去了什么筹码？ | 钱？时间？机会成本？ |
| 他们现在处于什么阶段（焦虑/止损/观望）？ | 判断是否在引爆点附近 |
| 他们最怕什么？ | 找到不安的根源 |
| 他们最渴望什么状态？ | 找到欲望的画面 |

### Step 2：构建内容框架

每条视频遵循四段结构：

```
1. 【代价描述】0-5秒
   精准描述目标客户已经付出的代价
   "你说的越准，他越觉得这人在说我"
   -> 你就变成了他眼里的那个出口

2. 【引爆点确认】5-20秒
   让他意识到：你现在就处在必须解决的时刻
   不是未来，不是以后，是现在

3. 【双开关触发】20-60秒
   欲望画面：拥有之后的具体状态（极度具体）
   不安解法：跟着我这件事稳（确定感传递）
   两个开关组合使用

4. 【行动号召】60-75秒
   给出一个低门槛的下一步
   让他觉得：找你就是止损的出口
```

### Step 3：撰写口播稿

写作要求：

- 开头不聊专业，**先描述他付出的代价**
- 用"你是不是也..."句式建立"这人在说我"的感觉
- 欲望画面要具体到能看见、能感受到（不要抽象描述）
- 不安要用确定性对冲（数据、案例、结果、步骤）
- 不要催单，要让他自己推理出"我应该找你"
- 时长控制在 65-75 秒

### Step 4：输出格式

```
## 选题：[选题名称]

### 精准客户定位
- 目标人群：{who}
- 已投入筹码：{time/money/opportunity cost}
- 引爆点时刻：{what makes them must-solve-now}
- 欲望画面：{what they desperately want}
- 不安根源：{what they fear most}

### 口播稿（预计 {duration} 秒）
{script}

### 字幕版
{subtitle_lines}

### 标题候选
1. {title}
2. {title}
3. {title}
```
